# 如何为不同前端资源配置缓存策略？如何强制刷新特定资源？

### 一、Nginx 缓存策略配置

核心思路是根据**资源类型**（通过文件扩展名匹配）设置不同的 `Cache-Control` 或 `Expires` 响应头。

#### 1. 基础配置示例

通常，我们将资源分为以下几类，并为它们设置不同的缓存策略：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/your/webroot;

    # 1. 对于 HTML 文件：完全不缓存或极短时间缓存，确保用户总能获取最新版本
    location ~* \.html$ {
        add_header Cache-Control "no-cache, no-store, must-revalidate";
        # 或者更严格的：add_header Cache-Control "no-cache";
        # 或者针对单页应用的入口文件，通常设置为不缓存
    }

    # 2. 对于 CSS、JavaScript、Web字体等频繁变动的资源
    # 使用「缓存破坏」策略，通过文件名中的哈希值（如 app.a1b2c3.css）来强制更新
    # 因为这些文件内容变化后URL会变，所以可以设置长期缓存
    location ~* \.(css|js|woff2?|ttf|eot)$ {
        expires 1y;  # 缓存1年
        add_header Cache-Control "public, immutable";
    }

    # 3. 对于图片等静态媒体资源
    location ~* \.(jpg|jpeg|png|gif|ico|svg|webp)$ {
        expires 6M;  # 缓存6个月
        add_header Cache-Control "public";
    }

    # 4. 对于版本化资源（URL中带哈希值，如 /dist/app.abc123.js）
    # 这是最安全的长期缓存策略
    location ~* \.[a-f0-9]{8,}\.(css|js)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

#### 2. 配置详解

- **`expires`**：Nginx指令，设置资源的过期时间。
  - `1y`：1年
  - `6M`：6个月  
  - `1d`：1天
  - `-1`：不缓存

- **`Cache-Control` 响应头**：
  - `public`：允许浏览器和CDN等中间代理缓存
  - `private`：仅允许浏览器缓存
  - `no-cache`：可以缓存，但每次使用前必须向服务器验证新鲜度
  - `no-store`：完全不缓存
  - `must-revalidate`：缓存必须在使用前验证
  - `immutable`：表示资源内容永不变，浏览器不会发送重新验证请求（特别适合哈希版本化资源）
  - `max-age=31536000`：缓存最大有效时间（秒）

### 二、强制刷新特定资源的策略

#### 1. 基于文件名哈希的「缓存破坏」

这是现代前端构建工具（Webpack、Vite等）的标配方案，也是**最推荐的方案**。

**原理**：当文件内容变化时，生成新的文件名（通过添加哈希值）。

- **构建前**：`app.js`、`style.css`
- **构建后**：`app.a1b2c3.js`、`style.d4e5f6.css`

由于URL完全不同，浏览器会将其视为**全新的资源**，自然就会下载最新版本。

**Nginx配置**（配合构建工具）：
```nginx
# 对带哈希版本的文件设置超长缓存
location ~* \.[a-f0-9]{8,}\.(css|js|woff2?)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
}

# 入口HTML文件不缓存或极短缓存
location / {
    try_files $uri $uri/ /index.html;
    # 如果是SPA，所有路由都指向index.html
    
    add_header Cache-Control "no-cache";
}
```

#### 2. 查询字符串版本号（传统方案）

在资源URL后添加版本号参数：
- `app.js?v=1.2.3`
- `style.css?version=20240101`

**Nginx配置**：
```nginx
# 注意：有些代理服务器可能默认忽略查询字符串，需要明确配置
location ~* \.(css|js)$ {
    # 确保Nginx正确处理带查询字符串的相同文件
    expires 1y;
    add_header Cache-Control "public";
}
```

⚠️ **注意**：这种方法不如文件名哈希可靠，因为某些中间代理可能忽略查询字符串。

#### 3. 手动清除Nginx代理缓存

如果你使用了Nginx作为**反向代理并开启了缓存**，可能需要清除代理缓存：

```nginx
# 在http块中配置代理缓存路径
http {
    proxy_cache_path /path/to/cache levels=1:2 keys_zone=my_cache:10m;
    
    server {
        location / {
            proxy_cache my_cache;
            proxy_pass http://backend;
            
            # 可以通过特定请求头或URL参数来绕过缓存
            # 例如在URL中添加?nocache=1来跳过缓存
            if ($arg_nocache) {
                set $proxy_cache_bypass 1;
            }
        }
    }
}
```

**清除缓存方法**：
```bash
# 删除整个缓存目录
sudo rm -rf /path/to/cache/*

# 或者使用Nginx的缓存清除功能（需要额外配置）
```

### 三、完整的最佳实践配置示例

```nginx
server {
    listen 80;
    server_name example.com;
    root /var/www/dist;
    index index.html;

    # SPA应用：所有路由指向index.html
    location / {
        try_files $uri $uri/ /index.html;
        
        # HTML文件不缓存（或很短缓存）
        add_header Cache-Control "no-cache";
    }

    # 版本化资源（带8位以上哈希值） - 长期缓存
    location ~* \.[a-f0-9]{8,}\.(css|js)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # 普通CSS/JS资源
    location ~* \.(css|js)$ {
        expires 1d;  # 非版本化的设置较短缓存
        add_header Cache-Control "public";
    }

    # 图片资源
    location ~* \.(jpg|jpeg|png|gif|ico|svg|webp)$ {
        expires 6M;
        add_header Cache-Control "public";
    }

    # Web字体
    location ~* \.(woff2?|ttf|eot)$ {
        expires 1y;
        add_header Cache-Control "public";
    }

    # 提供一种强制刷新的方式：在URL后添加?refresh=true
    if ($arg_refresh) {
        add_header Cache-Control "no-cache";
        add_header X-Refresh "forced";
    }
}
```

### 总结

| 资源类型 | 缓存策略 | 刷新机制 |
|---------|----------|----------|
| **HTML入口文件** | `no-cache` 或很短缓存 | 浏览器普通刷新 |
| **版本化资源**（带哈希） | 长期缓存（1年+`immutable`） | 修改文件名（构建时自动完成） |
| **CSS/JS/字体** | 长期缓存 | 使用文件名哈希 |
| **图片等媒体** | 中期缓存（数月） | 修改URL或等待过期 |
| **开发调试** | 不缓存 | 浏览器强制刷新（Ctrl+F5） |

**最佳实践组合**：
1. **使用前端构建工具生成哈希文件名**
2. **HTML入口文件设置`no-cache`**
3. **版本化资源设置长期缓存+`immutable`**
4. **非版本化资源设置较短缓存**

这样既保证了加载性能，又能确保用户及时获取更新。
