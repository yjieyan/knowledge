# 函数式组件与函数式编程思想

>「用函数写 UI，用代数思维管理副作用」——这就是现代 React 的全部哲学。

---

## 1 什么是函数式组件（FC）

| 版本 | 组件写法 | 特点 |
|----|---------|------|
| React ≤ 16.7 | `class MyComp extends React.Component` | 生命周期 + this |
| React ≥ 16.8 | `const MyComp = (props) => <div/>` | 无实例、无 this、支持 Hooks |

### 1.1 最小 FC
```tsx
type GreetingProps = { name: string };
const Greeting: React.FC<GreetingProps> = ({ name }) => <h1>Hello {name}</h1>;
```

### 1.2 与 class 组件差异速览
| 能力 | FC + Hooks | class |
|---|---|---|
| state | useState / useReducer | this.state |
| 生命周期 | useEffect | 多个生命周期函数 |
| this 指向 | 无 | 有 |
| 性能优化 | React.memo + useMemo | shouldComponentUpdate |
| 代码复用 | 自定义 Hook | HOC / Render Prop |

---

## 2 函数式编程（FP）核心概念

> 记忆口诀：**「纯函、不可、组合、高阶、惰性、幺半」**

| 概念 | 一句话 | React 映射 |
|----|--------|------------|
| **纯函数** | 相同输入必相同输出，无副作用 | 组件 = f(props) |
| **不可变** | 不修改原值，返回新值 | setState 必须新对象 |
| **高阶函数** | 函数入参或返回是函数 | React.memo / useCallback |
| **组合** | 小函数拼成大函数 | 组件组合：`<A><B/></A>` |
| **柯里化** | 多参转单参嵌套 | hooks 链式调用 |
| **函子** | 盒子 map 不拆盒 | Array.map / Promise.then |
| **惰性** | 延迟计算 | Suspense / lazy |

## 3 从代码看 FP 思想

### 3.1 纯组件 = 纯函数
```tsx
// 纯：无外部变量、无网络、无 Date.now()
const Avatar = ({ uid, src }: { uid: string; src: string }) => (
  <img src={src} alt={uid} width={48} height={48} />
);
```

### 3.2 不可变更新
```tsx
// ❌ 突变
const next = state; 
next.count++;

// ✅ 新对象
setState(prev => ({ ...prev, count: prev.count + 1 }));
```

### 3.3 高阶组件 & 高阶函数
```tsx
// HOC：函数返回组件
const withLoading = <P extends object>(Component: React.FC<P>) =>
  ({ isLoading, ...props }: { isLoading: boolean } & P) =>
    isLoading ? <Spinner/> : <Component {...props as P} />;

// 使用
const UserTable = withLoading(BaseTable);
```

### 3.4 组合 > 继承
```tsx
// 用组件拼组件，而非 extends
const Page = () => (
  <Layout header={<Header />} sidebar={<Menu />}>
    <Content />
  </Layout>
);
```

### 3.5 惰性加载
```tsx
const LazyChart = lazy(() => import('./HeavyChart'));

<Suspense fallback={<Skeleton />}>
  <LazyChart />
</Suspense>
```

## 4 Hooks = FP 在 React 的语法糖

| Hook | FP 意义 |
|----|--------|
| useState | 状态是函子，setState 是 map |
| useReducer | 代数数据类型（Action）+ 纯函数 reduce |
| useMemo | 记忆化（缓存纯函数结果） |
| useCallback | 函数引用相等性优化 |
| useRef | 副作用容器（不破坏纯函数外观） |
| useEffect | 把副作用推到 Monad 边缘 |

### 4.1 自定义 Hook —— 函数组合终极形态
```tsx
// 封装数据获取 & 取消
function useFetch<T>(url: string) {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    const ctl = new AbortController();
    fetch(url, { signal: ctl.signal })
      .then(r => r.json())
      .then(setData)
      .finally(() => setLoading(false));
    return () => ctl.abort();
  }, [url]);

  return { data, loading };
}
```

## 5 代数思维：把业务写成 reduce

```tsx
type Action =
  | { type: 'inc' }
  | { type: 'dec' }
  | { type: 'add'; payload: number };

const reducer = (s: number, a: Action): number =>
  a.type === 'inc'   ? s + 1
: a.type === 'dec'   ? s - 1
: a.type === 'add'   ? s + a.payload
: s;

function Counter() {
  const [count, dispatch] = useReducer(reducer, 0);
  return (
    <>
      <button onClick={() => dispatch({ type: 'dec' })}>-</button>
      {count}
      <button onClick={() => dispatch({ type: 'inc' })}>+</button>
      <button onClick={() => dispatch({ type: 'add', payload: 10 })}>+10</button>
    </>
  );
}
```

## 6 性能：记忆化 = 函数缓存

```tsx
const ExpensiveList = React.memo(function List({ items, filter }) {
  const visible = useMemo(() => items.filter(filter), [items, filter]);
  return <ul>{visible.map(v => <li key={v.id}>{v.name}</li>)}</ul>;
});
```

## 7 副作用边界：把不纯推出去

```tsx
// 纯函数内部无 IO
const sendMessage = (text: string): Promise<string> =>
  fetch('/api/chat', { method: 'POST', body: text }).then(r => r.json());

// 在组件边缘调用
const Chat = () => {
  const [msg, setMsg] = useState('');
  const { run, loading } = useAsync(() => sendMessage(msg)); // 自定义 Hook
  return (
    <>
      <input value={msg} onChange={e => setMsg(e.target.value)} />
      <button onClick={run} disabled={loading}>发送</button>
    </>
  );
};
```

## 8 常见误区与纠正

| 误区 | 正确做法 |
|----|----------|
| 在 FC 里直接修改 props | props 只读，克隆后返回新对象 |
| useState 里保存计算结果 | 用 useMemo 缓存 |
| 把函数写在 render 里 | 用 useCallback 固化引用 |
| 用 ref 存“经常变”的 UI 值 | ref 不触发重渲染，只放副作用或缓存 |

## 9 进阶：函子 & Monad（拓展阅读）

```tsx
// Array 是函子
[1, 2, 3].map(x => x * 2); // [2, 4, 6]

// Promise 是 Monad
Promise.resolve(1)
  .then(x => x + 1)   // 扁平化
  .then(x => console.log(x)); // 2
```

React 的「 Suspense → ErrorBoundary → retry」可视为 **Monad 错误处理**。
